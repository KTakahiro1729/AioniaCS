# キャラクターシートのセッションストレージ移行メモ

## 変更概要
- ローカル保存先を `localStorage` から `sessionStorage` に変更し、タブ（ブラウザセッション）ごとに編集データが独立するようにしました。
- `useLocalCharacterPersistence` の既存のデバウンス処理や共有モード判定ロジックには影響を与えていません。

## 動作確認手順
1. キャラクターシートを開いて任意の値を編集する。
2. ブラウザをリロードし、編集内容が復元されることを確認する（同一タブ内では `sessionStorage` が維持される）。
3. 同じアプリを新しいタブで開き、フィールドが初期状態であることを確認する。
4. 片方のタブで編集した値が、もう一方のタブに影響しないことを確認する。
5. 共有モード（`isViewingShared` が true になる操作）では保存が行われないことを確認する。
